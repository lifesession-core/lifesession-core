# Contributing

Thanks for helping improve LifeSession Core.

## Development Principles

- Keep the public package useful without exposing private operations.
- Prefer schemas, examples, docs, and small demo components.
- Use fake data in examples.
- Keep simulation code clearly labeled and disconnected from real services.

## Pull Request Checklist

- No secrets, tokens, internal URLs, private paths, or real user data.
- No imports from private repositories or internal packages.
- No live execution logic.
- Tests or examples updated when behavior changes.
- Documentation updated for user-facing changes.
